## 2. Supabase 데이터베이스 세팅

### 2.1 Supabase 프로젝트 생성

1. [https://supabase.com](https://supabase.com) 접속 → 로그인
2. **New Project** 클릭
3. 프로젝트명: `patent-platform`, 지역: `Northeast Asia (Seoul)` 선택
4. 비밀번호 설정 후 **Create new project**

### 2.2 데이터베이스 스키마 생성

Supabase Dashboard → **SQL Editor**에서 아래 SQL 실행:

```sql
-- 1. 출원인 테이블
CREATE TABLE IF NOT EXISTS applicants (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          TEXT NOT NULL,
  name_en       TEXT,
  country       TEXT NOT NULL DEFAULT 'KR',
  type          TEXT NOT NULL CHECK (type IN ('corporation', 'individual')),
  raw_json      JSONB,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 2. 특허 테이블
CREATE TABLE IF NOT EXISTS patents (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_patent_id    TEXT UNIQUE NOT NULL,
  title               TEXT NOT NULL,
  title_en            TEXT,
  abstract            TEXT,
  applicant_id        UUID REFERENCES applicants(id),
  applicant_name      TEXT,
  inventor_names      TEXT[] NOT NULL DEFAULT '{}',
  ipc_codes           TEXT[] NOT NULL DEFAULT '{}',
  ipc_names           TEXT[] NOT NULL DEFAULT '{}',
  filing_date         DATE,
  publication_date    DATE,
  registration_date   DATE,
  expiry_date         DATE,
  status              TEXT NOT NULL CHECK (status IN ('PENDING','REGISTERED','EXPIRED','REJECTED','WITHDRAWN')),
  source              TEXT NOT NULL CHECK (source IN ('KIPRIS','USPTO')),
  raw_json            JSONB,
  created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 3. 조직 테이블
CREATE TABLE IF NOT EXISTS orgs (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL,
  plan        TEXT NOT NULL DEFAULT 'free' CHECK (plan IN ('free','pro','enterprise')),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 4. 조직 멤버 테이블
CREATE TABLE IF NOT EXISTS org_members (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  role        TEXT NOT NULL DEFAULT 'member' CHECK (role IN ('admin','member')),
  UNIQUE (org_id, user_id)
);

-- 5. 주시 목록 테이블
CREATE TABLE IF NOT EXISTS watchlist (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  patent_id   UUID NOT NULL REFERENCES patents(id) ON DELETE CASCADE,
  note        TEXT,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (user_id, patent_id)
);

-- 6. 알림 규칙 테이블
CREATE TABLE IF NOT EXISTS alert_rules (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      UUID NOT NULL REFERENCES orgs(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,
  type        TEXT NOT NULL CHECK (type IN ('COMPETITOR','IPC','EXPIRY','KEYWORD')),
  rule_json   JSONB NOT NULL DEFAULT '{}',
  channel     TEXT NOT NULL DEFAULT 'EMAIL' CHECK (channel IN ('EMAIL','KAKAO')),
  is_enabled  BOOLEAN NOT NULL DEFAULT true,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 7. 알림 로그 테이블
CREATE TABLE IF NOT EXISTS alert_logs (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  alert_rule_id   UUID NOT NULL REFERENCES alert_rules(id) ON DELETE CASCADE,
  patent_id       UUID NOT NULL REFERENCES patents(id) ON DELETE CASCADE,
  sent_at         TIMESTAMPTZ NOT NULL DEFAULT now(),
  status          TEXT NOT NULL CHECK (status IN ('SENT','FAIL')),
  error_message   TEXT
);
```

### 2.3 인덱스 생성

```sql
CREATE INDEX IF NOT EXISTS idx_patents_status ON patents(status);
CREATE INDEX IF NOT EXISTS idx_patents_source ON patents(source);
CREATE INDEX IF NOT EXISTS idx_patents_filing_date ON patents(filing_date);
CREATE INDEX IF NOT EXISTS idx_patents_expiry_date ON patents(expiry_date);
CREATE INDEX IF NOT EXISTS idx_patents_applicant_name ON patents(applicant_name);
CREATE INDEX IF NOT EXISTS idx_watchlist_user_id ON watchlist(user_id);
CREATE INDEX IF NOT EXISTS idx_alert_rules_user_id ON alert_rules(user_id);
CREATE INDEX IF NOT EXISTS idx_alert_logs_sent_at ON alert_logs(sent_at);
```

### 2.4 Row Level Security (RLS) 설정

```sql
-- 전체 테이블 RLS 활성화
ALTER TABLE patents ENABLE ROW LEVEL SECURITY;
ALTER TABLE watchlist ENABLE ROW LEVEL SECURITY;
ALTER TABLE alert_rules ENABLE ROW LEVEL SECURITY;
ALTER TABLE alert_logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE orgs ENABLE ROW LEVEL SECURITY;
ALTER TABLE org_members ENABLE ROW LEVEL SECURITY;

-- 특허: 누구나 읽기 가능
CREATE POLICY "patents_public_read" ON patents FOR SELECT USING (true);
-- 특허: service_role만 쓰기 (크론 잡)
CREATE POLICY "patents_service_insert" ON patents FOR INSERT WITH CHECK (auth.role() = 'service_role');
CREATE POLICY "patents_service_update" ON patents FOR UPDATE USING (auth.role() = 'service_role');

-- 주시 목록: 본인만 CRUD
CREATE POLICY "watchlist_user_all" ON watchlist
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- 알림 규칙: 본인만 CRUD
CREATE POLICY "alert_rules_user_all" ON alert_rules
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- 알림 로그: 본인 규칙의 로그만 조회
CREATE POLICY "alert_logs_user_read" ON alert_logs FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM alert_rules
      WHERE alert_rules.id = alert_logs.alert_rule_id
        AND alert_rules.user_id = auth.uid()
    )
  );
```

## 3. 환경 변수 관리

### 3.1 `.env.local` 전체 항목

```bash
# ──────────────────────────────────────────
# 목업 모드 (true: Supabase 없이 샘플 데이터)
# ──────────────────────────────────────────
NEXT_PUBLIC_MOCK=false

# ──────────────────────────────────────────
# Supabase
# ──────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...   # 크론 잡 전용, 절대 클라이언트 노출 금지

# ──────────────────────────────────────────
# KIPRIS 특허 공공 API
# ──────────────────────────────────────────
KIPRIS_API_KEY=<공공데이터포털_발급_키>
KIPRIS_API_BASE_URL=https://plus.kipris.or.kr/openapi/rest

# ──────────────────────────────────────────
# Resend 이메일 발송
# ──────────────────────────────────────────
RESEND_API_KEY=re_xxxxxxxxxxxx
RESEND_FROM_EMAIL=noreply@your-domain.com

# ──────────────────────────────────────────
# 크론 잡 인증 (32자 이상 랜덤 문자열)
# ──────────────────────────────────────────
CRON_SECRET=your-random-secret-at-least-32-chars

# ──────────────────────────────────────────
# 앱 URL
# ──────────────────────────────────────────
NEXT_PUBLIC_APP_URL=https://your-domain.com
```

### 3.2 환경 변수 발급 방법

**KIPRIS API Key**
1. [공공데이터포털](https://www.data.go.kr) 회원가입
2. "특허청_키프리스 특허실용신안 검색서비스" 검색 → 활용 신청
3. 마이페이지 → 인증키 발급 (일반 인증키)

**Resend API Key**
1. [https://resend.com](https://resend.com) 가입
2. API Keys → Create API Key

**CRON_SECRET 생성**
```bash
# PowerShell
-join ((65..90) + (97..122) + (48..57) | Get-Random -Count 40 | ForEach-Object {[char]$_})
# 또는 Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

## 4. 배포 (Vercel)

### 4.1 Vercel 초기 배포

```bash
# Vercel CLI 설치
npm i -g vercel

# 로그인
vercel login

# 배포 (프로젝트 루트에서)
vercel

# 프로덕션 배포
vercel --prod
```

### 4.2 Vercel 환경 변수 설정

Vercel Dashboard → 프로젝트 → **Settings → Environment Variables**에서 `.env.local`의 모든 항목을 입력합니다.

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY`는 반드시 **Production/Preview**에만 설정하고, **NEXT_PUBLIC_** 접두사가 붙지 않도록 주의합니다.

### 4.3 vercel.json (크론 잡 설정)

프로젝트 루트에 `vercel.json` 파일을 생성합니다:

```json
{
  "crons": [
    {
      "path": "/api/jobs/poll-patents",
      "schedule": "0 17 * * *"
    },
    {
      "path": "/api/jobs/process-alerts",
      "schedule": "0 23 * * *"
    }
  ]
}
```

> 시간대 주의: Vercel Cron은 UTC 기준입니다.  
> `0 17 * * *` = 한국 시간 오전 2시(UTC+9)  
> `0 23 * * *` = 한국 시간 오전 8시(UTC+9)

## 5. 크론 잡 운영

### 5.1 특허 수집 잡 (`/api/jobs/poll-patents`)

**역할**: KIPRIS에서 어제~오늘 출원된 신규 특허를 수집해 DB에 upsert합니다.

**수동 실행 방법**:
```bash
curl -X POST https://your-domain.com/api/jobs/poll-patents \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

**응답 예시**:
```json
{
  "success": true,
  "data": {
    "inserted": 42,
    "updated": 3,
    "errors": []
  }
}
```

### 5.2 알림 처리 잡 (`/api/jobs/process-alerts`)

**역할**: 활성화된 모든 알림 규칙을 평가하여 조건에 맞는 특허 발견 시 이메일/카카오 알림을 발송합니다.

**수동 실행 방법**:
```bash
curl -X POST https://your-domain.com/api/jobs/process-alerts \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

### 5.3 크론 잡 실패 대응

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 401 Unauthorized | CRON_SECRET 불일치 | 환경 변수 확인 |
| KIPRIS API 오류 | API 키 만료 또는 일일 한도 초과 | 공공데이터포털에서 키 갱신 |
| Supabase 연결 실패 | DB 과부하 또는 키 만료 | Supabase 대시보드 확인 |
| 이메일 발송 실패 | Resend 한도 초과 | Resend 대시보드 확인 |

## 6. 사용자·조직 관리

### 6.1 신규 사용자 가입 흐름

1. 사용자가 `/login` → **회원가입** 탭에서 이메일/비밀번호 입력
2. Supabase Auth 이메일 인증 메일 발송 (자동)
3. 사용자가 인증 링크 클릭 → 가입 완료
4. 로그인 후 자동으로 개인 조직(org) 생성 (자동화 트리거 필요)

### 6.2 Supabase Dashboard에서 사용자 관리

**Path**: Supabase Dashboard → Authentication → Users

| 작업 | 방법 |
|------|------|
| 사용자 목록 조회 | Authentication → Users |
| 강제 이메일 인증 | 사용자 클릭 → Confirm email |
| 사용자 비활성화 | 사용자 클릭 → Ban user |
| 비밀번호 초기화 | 사용자 클릭 → Send password reset |

### 6.3 조직 플랜 변경 (SQL)

```sql
-- Pro로 업그레이드
UPDATE orgs SET plan = 'pro' WHERE id = 'ORG_UUID';

-- Enterprise로 업그레이드
UPDATE orgs SET plan = 'enterprise' WHERE id = 'ORG_UUID';
```

### 6.4 조직 멤버 추가 (SQL)

```sql
INSERT INTO org_members (org_id, user_id, role)
VALUES ('ORG_UUID', 'USER_UUID', 'member');
```

## 7. 목업 모드 전환

### 7.1 목업 모드 활성화

`.env.local`:
```bash
NEXT_PUBLIC_MOCK=true
```

- Supabase 연결 없이 `src/lib/mock-data.ts`의 샘플 데이터 사용
- 12개 특허, 3개 주시 항목, 3개 알림 규칙 기본 제공
- 로그인 없이 바로 대시보드 접근 가능 (데모·영업용)

### 7.2 목업 모드 비활성화

`.env.local`:
```bash
NEXT_PUBLIC_MOCK=false
# 또는 해당 줄 삭제
```

실제 Supabase URL·키 입력 후 서버 재시작:
```bash
# node 프로세스 종료 후
npm run dev
```

### 7.3 목업 데이터 수정

`src/lib/mock-data.ts` 파일에서 직접 수정합니다:
- `MOCK_PATENTS`: 특허 목록 (12건 기본)
- `MOCK_WATCHLIST`: 주시 목록
- `MOCK_ALERT_RULES`: 알림 규칙
- `MOCK_ALERT_LOGS`: 알림 이력
- `MOCK_REPORT`: 분석 리포트 데이터

## 8. 로그 및 모니터링

### 8.1 서버 로그 확인

**Vercel 배포 시**: Vercel Dashboard → Functions → 함수 클릭 → Logs

**로컬 개발 시**: 터미널에서 `npm run dev` 실행 화면 관찰

### 8.2 API 응답 구조

모든 API는 다음 형식으로 응답합니다:

```json
// 성공
{ "success": true, "data": {...} }

// 실패
{ "success": false, "message": "오류 설명", "errors": {...} }
```

**HTTP 상태 코드**:
| 코드 | 의미 |
|------|------|
| 200 | 성공 |
| 201 | 생성 성공 |
| 400 | 유효성 검증 실패 |
| 401 | 인증 필요 |
| 403 | 권한 없음 |
| 404 | 리소스 없음 |
| 500 | 서버 내부 오류 |

### 8.3 헬스체크 엔드포인트

```bash
curl https://your-domain.com/api/health
# → { "success": true, "data": { "status": "ok", "timestamp": "..." } }
```

### 8.4 Supabase 모니터링

- **Database 사용량**: Supabase Dashboard → Settings → Usage
- **실시간 쿼리 로그**: Supabase Dashboard → Database → Logs
- **인증 이벤트**: Supabase Dashboard → Authentication → Logs

## 10. 트러블슈팅

### 문제 1: `.next/dev/lock` 오류로 서버 시작 실패

```
Unable to acquire lock at .next/dev/lock
```

**해결**:
```powershell
# Windows PowerShell
Get-Process node -ErrorAction SilentlyContinue | Stop-Process -Force
Remove-Item ".next/dev/lock" -ErrorAction SilentlyContinue
npm run dev
```

### 문제 2: Supabase 클라이언트 초기화 오류

```
Your project's URL and Key are required to create a Supabase client!
```

**원인**: `.env.local`에 Supabase 키가 없거나 목업 모드가 꺼져 있음  
**해결**: `.env.local`에 `NEXT_PUBLIC_MOCK=true` 설정 또는 실제 Supabase 키 입력

### 문제 3: proxy.ts / middleware.ts 충돌

```
Both middleware file and proxy file are detected.
```

**해결**: `src/middleware.ts` 파일 삭제 (`src/proxy.ts`만 유지)

### 문제 4: KIPRIS API 응답 없음

**원인**: API 키 만료, 일일 호출 한도 초과, 네트워크 오류  
**확인**:
```bash
curl "https://plus.kipris.or.kr/openapi/rest/patUtiModInfoSearchSevice/patentUtilityInfo?serviceKey=YOUR_KEY&type=json&numOfRows=1"
```

### 문제 5: TypeScript 컴파일 오류

```bash
npx tsc --noEmit
```
오류가 없으면 종료 코드 0. 오류 발생 시 해당 파일·라인 확인.
