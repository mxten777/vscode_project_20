## 1. 서비스 소개

**PatentPlatform**은 한국 특허청 데이터를 기반으로 특허를 검색하고, 관심 특허를 추적하며, 경쟁사의 새로운 특허 출원을 자동으로 알려주는 서비스입니다.

### 주요 기능 한눈에 보기

| 기능 | 설명 |
|------|------|
| 🔍 **특허 검색** | 키워드·출원인·IPC 코드 등으로 특허 검색 |
| ❤️ **주시 목록** | 관심 특허 저장 및 메모 관리 |
| 🔔 **알림 규칙** | 경쟁사·기술 분야·만료 임박 자동 알림 |
| 📊 **분석 리포트** | 기간별 출원 추이·상태 분포·출원인 순위 |

## 2. 시작하기

### 2.1 회원가입

1. 브라우저에서 서비스 주소 접속
2. **회원가입** 탭 클릭
3. 이메일 주소와 비밀번호 입력

   > ⚠️ 비밀번호는 최소 8자 이상으로 설정하세요.

4. **회원가입** 버튼 클릭
5. 입력한 이메일로 인증 메일이 발송됩니다.
6. **이메일 인증 링크** 클릭 후 로그인 가능

### 2.2 로그인

1. 이메일과 비밀번호 입력
2. **로그인** 버튼 클릭

### 2.3 로그아웃

화면 우측 상단 **사용자 아이콘** → **로그아웃**

## 3. 특허 검색 및 조회

메인 화면(`/`)에서 전체 특허를 검색하고 필터링할 수 있습니다.

### 3.1 기본 검색

화면 상단 검색창에 키워드를 입력하고 **Enter**를 누릅니다.

검색 대상: **특허 제목**, **출원인명**

```
예시: "인공지능" — AI 관련 특허 검색
     "삼성전자" — 삼성전자 출원 특허 검색
```

### 3.2 필터 사용

검색창 우측 **필터(☰)** 버튼을 클릭하면 상세 필터가 펼쳐집니다.

| 필터 | 설명 | 예시 |
|------|------|------|
| **상태** | 출원중/등록/만료/거절/취하 | 등록 → 등록 완료된 특허만 |
| **출처** | KIPRIS / USPTO | KIPRIS → 한국 특허만 |
| **IPC 코드** | 기술 분류 코드 | H01L21/66 |
| **출원인** | 특정 기업·개인 | LG에너지솔루션 |
| **발명자** | 발명자 이름 | 홍길동 |
| **출원일 범위** | 출원일 시작~종료 | 2024-01-01 ~ 2024-12-31 |

### 3.3 정렬

우측 상단 **정렬 기준** 드롭다운에서 선택합니다:
- 등록일순 (기본)
- 출원일순
- 만료일순
- 등록일순

**▲/▽** 버튼으로 오름차순/내림차순을 전환합니다.

### 3.4 페이지 이동

목록 하단의 **< 이전 / 다음 >** 버튼을 사용합니다.

기본 페이지당 20건이 표시됩니다.

## 4. 특허 상세 페이지

특허 목록에서 **특허 제목을 클릭**하면 상세 페이지로 이동합니다.

### 4.1 표시 정보

| 항목 | 설명 |
|------|------|
| **출원번호** | 특허청 출원 番号 |
| **상태** | 출원중/등록/만료/거절/취하 |
| **출원인** | 기업 또는 개인 이름 |
| **발명자** | 발명자 전체 목록 |
| **IPC 코드** | 기술 분류 코드 및 명칭 |
| **출원일** | 특허 출원 날짜 |
| **공개일** | 공개 날짜 |
| **등록일** | 등록 허가 날짜 |
| **만료일** | 특허 효력 만료 날짜 |
| **만료까지** | 오늘 기준 잔여 일수 |
| **초록** | 특허 내용 요약 |

### 4.2 주시 등록/해제

상세 페이지 우측 상단의 **❤️ 주시하기** 버튼을 클릭합니다.
- 이미 주시 중이면 **💔 주시 해제** 버튼이 표시됩니다.

## 5. 주시 목록 관리

상단 메뉴 **주시 목록** 또는 `/watchlist` 접속

### 5.1 주시 추가

특허 목록 또는 상세 페이지에서 **❤️ 버튼**을 클릭하면 주시 목록에 추가됩니다.

### 5.2 주시 카드 구성

각 카드에는 다음 정보가 표시됩니다:
- 특허 제목 및 출원번호
- 현재 상태 배지 (등록/출원중/만료 등)
- 만료 D-Day (등록된 특허만 표시)
- IPC 코드
- 주시 메모

### 5.3 메모 작성·수정

1. 카드 우측 **🗒 메모** 아이콘 클릭
2. 텍스트창에 메모 입력
3. **저장** 버튼 클릭

메모 예시:
```
- 침해 가능성 검토 필요 (2026 Q3)
- 법무팀 공유 완료
- 만료 후 기술 활용 검토
```

### 5.4 주시 해제

카드 우측 상단 **🗑 삭제** 아이콘 클릭 → 확인 다이얼로그에서 **삭제** 클릭

> ⚠️ 삭제 시 해당 특허의 메모도 함께 삭제됩니다.

## 6. 알림 규칙 설정

상단 메뉴 **알림 관리** 또는 `/alerts` 접속

### 6.1 알림 규칙 탭

**알림 규칙** 탭에서 규칙 목록을 확인하고 관리합니다.

### 6.2 새 알림 규칙 만들기

1. **+ 알림 규칙 추가** 버튼 클릭
2. 규칙 정보 입력:

**① 규칙 이름**
```
예: "삼성전자 반도체 모니터링"
```

**② 알림 유형 선택**

| 유형 | 언제 알림이 오나요? | 설정 방법 |
|------|-------------------|-----------|
| **경쟁사** | 특정 출원인이 새 특허를 출원할 때 | 출원인명 입력 |
| **IPC 코드** | 특정 기술 분야의 새 특허가 등록될 때 | IPC 코드 입력 (예: H01L21/66) |
| **만료 임박** | 주시 중인 특허가 만료되기 N일 전 | 알림 쿠션 일수 설정 |
| **키워드** | 제목·초록에 특정 단어가 포함된 특허가 나올 때 | 키워드 입력 |

**③ 알림 채널**
- **이메일**: 가입한 이메일로 발송
- **카카오알림톡**: 카카오 계정 연동 필요 (Pro 이상)

**④ 활성화 여부**: ON/OFF 토글

3. **저장** 버튼 클릭

### 6.3 알림 규칙 수정

규칙 카드 우측 **✏️ 수정** 버튼 클릭 → 내용 수정 → **저장**

### 6.4 알림 규칙 ON/OFF

규칙 카드의 **토글 스위치**를 클릭하여 일시 중지/재개할 수 있습니다.

### 6.5 알림 규칙 삭제

규칙 카드 우측 **🗑 삭제** 버튼 클릭 → 확인 다이얼로그에서 **삭제**

## 7. 알림 이력 확인

알림 관리 화면 → **알림 이력** 탭

최근 50건의 발송 기록이 표시됩니다.

| 컬럼 | 설명 |
|------|------|
| 발송 시각 | 알림이 발송된 일시 |
| 알림 규칙명 | 어떤 규칙이 트리거됐는지 |
| 관련 특허 | 알림이 발생한 특허 제목 |
| 상태 | ✅ 성공(SENT) / ❌ 실패(FAIL) |

## 8. 분석 리포트

상단 메뉴 **리포트** 또는 `/reports` 접속

### 8.1 기간 설정

화면 상단의 **기간 선택** 영역에서 시작일과 종료일을 캘린더로 선택합니다.

기본 기간: **최근 6개월**

### 8.2 요약 카드

| 카드 | 설명 |
|------|------|
| 전체 특허 | 기간 내 수집된 총 특허 건수 |
| 등록 특허 | 현재 등록 상태인 특허 건수 |
| 만료 임박 (30일) | 30일 내 만료되는 특허 건수 |
| 상태 분류 | 상태 구분 수 |

### 8.3 月별 출원 추이 (라인 차트)

- 가로축: 월 (YYYY-MM)
- 세로축: 출원 건수
- 마우스를 차트 위에 올리면 해당 월 수치 표시

### 8.4 상태별 분포 (파이 차트)

등록/출원중/만료/거절/취하 비율을 색상별로 표시합니다.

- 파란색: 등록 (REGISTERED)
- 초록색: 출원중 (PENDING)
- 빨간색: 만료 (EXPIRED)
- 노란색: 거절 (REJECTED)
- 보라색: 취하 (WITHDRAWN)

### 8.5 출원인 TOP 10 (가로 막대 차트)

기간 내 가장 많이 출원한 기업·개인 순위를 보여줍니다.

### 8.6 IPC 코드 TOP 5

기간 내 가장 많이 등장한 기술 분류 코드 순위를 보여줍니다.

## 10. 자주 묻는 질문 (FAQ)

### Q1. 특허 데이터는 얼마나 자주 업데이트되나요?
**A.** 매일 오전 2시에 KIPRIS에서 전날 출원된 신규 특허를 자동으로 수집합니다. 데이터는 하루 1회 업데이트됩니다.

### Q2. 검색 결과에 내가 찾는 특허가 없어요.
**A.** KIPRIS 공개 데이터만 수집되므로, 공개 전인 특허(출원일로부터 약 18개월 이전)는 검색되지 않습니다. 키워드 대신 **IPC 코드** 또는 **출원인명**으로 검색해 보세요.

### Q3. 알림이 오지 않아요.
**A.** 다음을 확인하세요:
1. 알림 규칙 탭에서 해당 규칙의 토글이 **ON**인지 확인
2. 스팸 메일함 확인 (이메일 알림)
3. 알림 이력 탭에서 발송 실패(FAIL) 여부 확인

### Q4. 만료 임박 알림은 언제 설정하는 게 좋은가요?
**A.** 일반적으로 다음 쿠션을 권장합니다:
- **90일**: 특허 갱신료 납부 검토 시작
- **30일**: 갱신 결정 마감
- **7일**: 최종 확인

알림 규칙 생성 시 `daysBeforeExpiry`에 `[90, 30, 7]`과 같이 복수 설정이 가능합니다.

### Q5. 주시 목록의 메모는 팀원과 공유되나요?
**A.** 현재 v1.0에서는 **개인 메모**로 동일 조직 내 다른 멤버에게는 보이지 않습니다. 향후 버전에서 팀 공유 메모 기능이 추가될 예정입니다.

### Q6. IPC 코드가 무엇인가요?
**A.** IPC(International Patent Classification)는 특허 기술을 분류하는 국제 표준 코드입니다.

```
예시:
H01L21/66 — 반도체 소자 검사/테스트
G06N3/04  — 인공 신경망
B25J9/16  — 로봇 제어 시스템
```

특허청 KIPRIS 홈페이지의 「IPC 분류 검색」에서 원하는 기술 분야의 코드를 찾을 수 있습니다.

### Q7. 같은 조직의 다른 사람을 초대하려면 어떻게 하나요?
**A.** 현재 v1.0에서는 관리자가 직접 DB에서 멤버를 추가합니다. v1.1에서 조직 초대 UI가 추가될 예정입니다. 멤버 추가가 필요하면 관리자에게 문의하세요.

### Q8. 비밀번호를 잊어버렸어요.
**A.** 로그인 화면에서 **비밀번호 찾기** 또는 관리자에게 재설정 요청을 합니다.

### Q9. Free 플랜과 Pro 플랜의 차이는 무엇인가요?
**A.** 
| 항목 | Free | Pro |
|------|------|-----|
| 주시 목록 | 최대 10건 | 무제한 |
| 알림 규칙 | 최대 2건 | 최대 20건 |
| 카카오알림톡 | ❌ | ✅ |
| 고급 리포트 | ❌ | ✅ |

### Q10. 데이터는 안전하게 관리되나요?
**A.** 
- 모든 데이터는 암호화된 연결(HTTPS)로 전송됩니다.
- 개인 데이터는 Supabase Row Level Security(RLS)로 다른 사용자와 완전히 분리됩니다.
- 비밀번호는 해시 처리되어 저장되며 원문은 보관하지 않습니다.
