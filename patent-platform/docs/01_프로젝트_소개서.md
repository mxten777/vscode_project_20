# PatentPlatform — 프로젝트 소개서

> **KIPRIS 기반 특허 모니터링 SaaS 플랫폼**  
> 버전 1.0 | 2026년 3월

---

## 1. 개요

**PatentPlatform**은 한국 특허청(KIPRIS) 공공데이터 API를 활용하여 특허 정보를 자동 수집·분석하고, 사용자에게 맞춤형 알림을 제공하는 B2B SaaS 서비스입니다.

기업 연구소·IP 담당자·스타트업이 방대한 특허 데이터를 빠르게 파악하고, 경쟁사 동향과 만료 특허를 선제적으로 추적할 수 있도록 설계되었습니다.

## 2. 배경 및 필요성

### 2.1 문제 인식

| 현황 | 문제점 |
|------|--------|
| 연간 국내 특허 출원 약 23만 건 | 수동 모니터링 한계 |
| IP 침해 소송 연간 수천 건 | 사전 탐지 부재 |
| 특허 만료 후 기술 공개 | 공개 기술 활용 기회 미인지 |
| 경쟁사 기술 동향 파악 어려움 | R&D 중복 투자 위험 |

### 2.2 기대 효과

- 특허 조사 시간 **90% 단축** (수동 3~4시간 → 자동 실시간)
- 만료 임박 특허 선제 알림으로 **라이선스·무효심판 전략** 수립 가능
- 경쟁사 출원 동향 자동 추적으로 **R&D 방향성** 검증

## 3. 핵심 기능

### 3.1 특허 검색 및 조회

- **키워드 검색**: 특허 제목·출원인 통합 검색
- **다중 필터**: 상태(출원중/등록/만료/거절/취하), 출처(KIPRIS/USPTO), IPC 코드, 출원인, 발명자, 출원일·만료일 범위
- **정렬**: 출원일·만료일·등록일·등록순 정렬 (오름차순/내림차순)
- **페이지네이션**: 페이지당 20건 기본, 최대 100건 설정 가능

### 3.2 주시 목록 (Watchlist)

- **원클릭 주시 등록/해제**: 특허 목록 또는 상세 페이지에서 즉시 등록
- **메모 기능**: 특허별 주시 이유·분석 메모 저장
- **주시 현황 카드**: 상태 배지, 만료 D-Day 표시

### 3.3 알림 규칙 (Alert Rules)

| 알림 유형 | 설명 | 설정 방법 |
|-----------|------|-----------|
| **경쟁사(COMPETITOR)** | 특정 출원인 신규 특허 모니터링 | 출원인명 입력 |
| **IPC 코드** | 특정 기술 분야 신규 특허 모니터링 | IPC 코드 입력 |
| **만료 임박(EXPIRY)** | 등록 특허 만료 N일 전 알림 | 알림 쿠션(90/30/7일 등) 설정 |
| **키워드(KEYWORD)** | 제목·초록 키워드 포함 특허 알림 | 키워드 입력 |

알림 채널: **이메일(Resend API)**, **카카오알림톡**

### 3.4 분석 리포트

- **월별 출원 추이 Line Chart**: 기간 설정 가능
- **상태별 분포 Pie Chart**: 등록/출원중/만료/거절/취하 비율
- **출원인 TOP 10 Bar Chart**: 기간 내 최다 출원인 순위
- **IPC 코드 Top 5**: 주력 기술 분야 분석
- **요약 KPI 카드**: 전체 특허 수, 등록 건수, 만료 임박 건수

### 3.5 자동화 크론 잡

| 잡 이름 | 엔드포인트 | 주기 | 역할 |
|---------|-----------|------|------|
| 특허 수집 | `POST /api/jobs/poll-patents` | 매일 02:00 | KIPRIS API 신규 특허 upsert |
| 알림 발송 | `POST /api/jobs/process-alerts` | 매일 08:00 | 알림 규칙 평가 → 이메일/카카오 발송 |

## 6. 데이터 모델

### 주요 테이블

| 테이블 | 설명 | 주요 컬럼 |
|--------|------|-----------|
| `patents` | 특허 정보 | source_patent_id, title, applicant_name, ipc_codes[], status, filing_date, expiry_date |
| `applicants` | 출원인 | name, name_en, country, type |
| `watchlist` | 사용자 주시 목록 | user_id, patent_id, note, org_id |
| `alert_rules` | 알림 규칙 | type(COMPETITOR/IPC/EXPIRY/KEYWORD), rule_json, channel, is_enabled |
| `alert_logs` | 알림 발송 이력 | alert_rule_id, patent_id, sent_at, status |
| `orgs` | 조직(기업) | name, plan(free/pro/enterprise) |
| `org_members` | 조직 멤버 | org_id, user_id, role(admin/member) |

## 8. 보안 설계

- **Supabase Auth**: JWT 기반 인증, 세션 쿠키 SSR 관리
- **Row Level Security (RLS)**: DB 레벨에서 조직·사용자별 데이터 격리
- **Cron Secret**: `CRON_SECRET` 환경변수, 타이밍 공격 방지 상수 시간 비교
- **Zod 유효성 검증**: 모든 API 입력 서버사이드 검증
- **CSRF 방지**: SameSite 쿠키 정책
