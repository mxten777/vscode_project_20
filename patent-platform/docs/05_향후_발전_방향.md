## 1. 현재 버전 현황 (v1.0)

### 핵심 기능 (완료)
- ✅ KIPRIS 특허 데이터 자동 수집 (일 1회 폴링)
- ✅ 특허 검색·필터·정렬·페이지네이션
- ✅ 주시 목록 + 개인 메모
- ✅ 알림 규칙 4종 (경쟁사/IPC/만료임박/키워드)
- ✅ 이메일 알림 발송 (Resend)
- ✅ 분석 리포트 (월별 추이·상태 분포·출원인 TOP10)
- ✅ 조직(Organization) 기반 멀티 테넌시
- ✅ RLS Row-Level Security 적용

### 알려진 한계
- 데이터: KIPRIS 국내 특허만 지원
- 알림 채널: 이메일만 지원 (카카오·Slack 미지원)
- 협업: 팀 메모 공유 불가, 조직 멤버 초대 UI 없음
- AI 기능: 없음
- API: 외부 공개 API 없음

## 2. 단기 개선 계획 (v1.1 — 2026 Q2)

### 2.1 조직 관리 UI

현재는 관리자가 DB에서 직접 멤버를 추가해야 합니다.

**목표 기능:**
```
Settings > Team
  ├── 멤버 초대 (이메일 → 인비테이션 링크)
  ├── 멤버 역할 변경 (admin / member)
  └── 멤버 제거
```

**구현 계획:**
- `organization_invitations` 테이블 추가
- `/api/org/invite` 엔드포인트 (Resend로 초대 메일 발송)
- `/join?token=...` 수락 페이지

### 2.2 카카오 알림톡 연동

이메일 알림은 열람율이 낮습니다. 모바일 카카오알림톡으로 전환 시 열람율이 크게 향상됩니다.

**구현 계획:**
- 카카오 비즈메시지 API 연동
- 알림 규칙 생성 시 채널 선택 (email / kakao / both)
- `alerts_logs` 테이블에 channel 컬럼 추가

### 2.3 팀 공유 메모

주시 목록의 메모를 조직 내 팀원과 공유합니다.

**구현 계획:**
- `notes` 테이블에 `is_shared: boolean` 컬럼 추가
- 공유 메모는 조직 전체 조회 가능 (RLS 조건 수정)
- 댓글 스레드 (1뎁스): `note_comments` 테이블

### 2.4 특허 요약 PDF 내보내기

분석 리포트 결과를 PDF로 내보낼 수 있습니다.

**구현 계획:**
- `@react-pdf/renderer` 라이브러리 활용
- `/api/reports/export?format=pdf` 엔드포인트
- 차트 스냅샷은 Canvas → Base64 삽입

## 3. 중기 발전 계획 (v1.2~v2.0 — 2026 Q3~2027 Q1)

### 3.1 AI 기반 특허 요약

특허 초록은 전문 용어가 많아 일반인이 이해하기 어렵습니다.

**목표:**
- 특허 상세 페이지에 **AI 한줄 요약** 표시
- 기술 분야를 쉬운 말로 설명하는 **3줄 설명** 자동 생성

**구현 계획:**
```typescript
// 특허 저장 시 요약 생성 (cron 또는 큐)
const summary = await openai.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{
    role: 'user',
    content: `다음 특허 초록을 한국어로 3줄로 요약해 주세요:\n\n${patent.abstract}`
  }]
})
```

- 요약 텍스트는 `patents.ai_summary` 컬럼에 저장 (캐시)
- 비용 최적화: GPT-4o-mini 사용, 초록 1200자 이상 시 트런케이트

### 3.2 관련 특허 추천

현재 특허와 기술적으로 유사한 특허를 추천합니다.

**구현 계획:**

```
[특허 초록] → [임베딩 생성 (text-embedding-3-small)]
            → [pgvector에 저장]
            → [코사인 유사도로 TOP 5 검색]
```

- Supabase `pgvector` 익스텐션 활성화
- `patent_embeddings` 테이블: `patent_id`, `embedding vector(1536)`
- 특허 상세 페이지 하단에 **"관련 특허"** 섹션 추가

### 3.3 침해 키워드 분석

사용자가 자신의 특허 기술을 입력하면, 잠재적으로 침해 가능성이 있는 특허 목록을 제시합니다.

**사용 흐름:**
```
사용자: "우리 기술 설명: [입력]"
  → 임베딩 생성
  → 유사도 TOP 20 특허 검색
  → GPT로 침해 가능성 판단 (높음/중간/낮음)
  → 결과 리스트 표시 (면책 문구 포함)
```

> ⚠️ 법적 자문이 아님을 명시하는 면책 문구 필수

### 3.4 USPTO 글로벌 특허 연동

미국 특허청(USPTO) 데이터를 추가합니다.

**데이터 수집 계획:**
- USPTO OData API (`api.patentsview.org`) 무료 사용
- `patents.source` 컬럼에 `'KIPRIS' | 'USPTO'` 구분 추가
- 크론 주기: 주 1회 (USPTO는 데이터 업데이트가 주간)

**UI 변화:**
- 검색 시 **출처** 필터 (한국 / 미국 / 전체)
- 특허 상세 페이지에 **원본 링크** (KIPRIS / Google Patents)

### 3.5 Slack / Teams 알림 연동

기업 고객의 핵심 요구사항 중 하나입니다.

**구현 계획:**
- Slack Incoming Webhook 연동
- Microsoft Teams Webhook 연동
- 알림 규칙 생성 시 Webhook URL 입력
- `alerts_logs`에 `channel: 'email' | 'slack' | 'teams' | 'kakao'`

## 4. 장기 비전 (v3.0 — 2027~2028)

### 4.1 모바일 앱 (React Native)

웹 서비스의 핵심 기능을 모바일 앱으로 제공합니다.

**타겟 플랫폼:** iOS 16+ / Android 12+

**핵심 기능:**
- 특허 검색 및 조회
- 주시 목록 관리
- 푸시 알림 (Firebase FCM)
- 분석 리포트 요약

**기술 스택:**
- React Native + Expo
- 백엔드는 현재 Next.js API Route 그대로 활용
- 인증은 Supabase Auth (동일 세션)

### 4.2 공개 REST API

파트너사·스타트업이 PatentPlatform 데이터를 자신의 서비스에 활용할 수 있도록 API를 공개합니다.

**예상 엔드포인트:**
```
GET /v1/patents?q=&status=&page=
GET /v1/patents/{id}
GET /v1/patents/{id}/similar     (유사 특허)
GET /v1/applicants/{name}/stats  (출원인 통계)
```

**인증:** API Key (Bearer Token) 방식

**요금제별 Rate Limit:**
| 플랜 | 요청/월 |
|------|---------|
| Free | 1,000 |
| Pro | 50,000 |
| Enterprise | 무제한 |

### 4.3 특허 맵 시각화

출원인·IPC 코드·연도별 특허 데이터를 **인터랙티브 맵**으로 시각화합니다.

**구현 형태:**
```
버블 차트: X=출원연도, Y=IPC 분류, 크기=건수
네트워크 그래프: 출원인 간 인용 관계
지역 히트맵: 발명자 소재지 기반 특허 밀도
```

**기술 스택:**
- D3.js 또는 Observable Plot
- React 서버 컴포넌트 + Suspense

### 4.4 AI 특허 가치 평가

특허의 상업적 가치를 AI로 추정합니다.

**평가 지수 구성:**
```
특허 가치 Score (0~100):
  ├── 인용 횟수 (후속 특허에서 인용된 횟수)
  ├── 청구항 수 (많을수록 넓은 권리)
  ├── 국제 출원 여부 (PCT/EP 포함 시 가점)
  ├── 출원인 규모 (대기업 vs 스타트업)
  └── 기술 분야 성장성 (IPC → 시장 성장률 매핑)
```

**활용:**
- 주시 목록에 "특허 가치"  점수 컬럼 추가
- M&A·라이선싱 의사결정 보조 도구

| 버전 | 기능 | 기술 |
|------|------|------|
| v1.2 | 특허 AI 요약 | GPT-4o-mini |

### 비용 관리 전략

AI 기능은 비용이 발생하므로 다음 전략을 적용합니다:

1. **캐시 우선**: 요약·임베딩은 DB에 저장, 동일 특허 재요청 시 API 호출 없음
2. **미니 모델 우선**: 요약은 `gpt-4o-mini` (1/10 비용)
3. **배치 처리**: 새 특허 임베딩은 크론 배치로 비동기 처리
4. **요금제 연동**: AI 심화 기능은 Pro 이상에서만 제공

## 8. 기술 부채 및 인프라 개선

### 8.1 데이터 수집 안정성

현재 KIPRIS polling은 단순 HTTP 요청으로 실패 시 재시도가 없습니다.

**개선 계획:**
- BullMQ (Redis 기반 큐) 도입으로 재시도·DLQ 처리
- 수집 실패 시 Slack 알림 (운영 채널)
- 수집 메트릭 대시보드 (Grafana Cloud 무료 티어)

### 8.2 풀텍스트 검색 성능

현재 `ILIKE '%keyword%'` 검색은 인덱스를 활용하지 못합니다.

**개선 계획:**
- PostgreSQL `tsvector` + `GIN 인덱스` 적용
- 또는 Typesense (경량 오픈소스 검색 엔진) 사이드카 운영

```sql
-- 개선 예시
ALTER TABLE patents ADD COLUMN fts_vector tsvector
  GENERATED ALWAYS AS (
    to_tsvector('korean', coalesce(title,'') || ' ' || coalesce(abstract,''))
  ) STORED;
CREATE INDEX idx_patents_fts ON patents USING GIN(fts_vector);
```

### 8.3 캐시 레이어

현재 리포트 요청은 매번 DB 집계 쿼리를 실행합니다.

**개선 계획:**
- Upstash Redis (서버리스 Redis)로 리포트 집계 결과 캐시 (TTL 1시간)
- Next.js `unstable_cache` 또는 `revalidateTag` 활용

### 8.4 E2E 테스트 추가

현재 테스트 커버리지: 0%

**목표:**
- Playwright로 핵심 사용자 플로우 E2E 테스트
  1. 로그인 → 검색 → 주시 등록
  2. 알림 규칙 생성 → 수정 → 삭제
  3. 리포트 기간 변경 → 차트 렌더링 확인
- GitHub Actions CI에 통합 (PR 시 자동 실행)

### 8.5 모니터링 & 에러 추적

현재 에러를 console.error로만 출력합니다.

**개선 계획:**
- Sentry 프리 티어 연동 (에러 추적, 성능 모니터링)
- Vercel Analytics 활성화 (Web Vitals)
- `/api/health` 헬스체크 엔드포인트 추가 (Uptime Robot)
